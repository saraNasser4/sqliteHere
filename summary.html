SELECT

It is for get something from something
  • SELECT trackid, name, composer, unitprice FROM tracks; You need to get x y z from db called tracks
  • SELECT * FROM tracks; You need to get all of the data from db called tracks

ORDER BY

since if you used the normal SELECT it won't be in order we use this, it's the last thing in the SELECT
  • SELECT trackid, name, albumid FROM tracks ORDER BY albumid ASC; You need to get x y z from db called tracks the ordering from the small to bigger
    • You can remove ASC since it's the default or you can add DESC
  • SELECT name, milliseconds, albumid FROM tracks ORDER BY albumid, milliseconds DESC; Here it will order using ASC in the albumid first and then will check for the milliseconds
  • SELECT name, milliseconds, albumid FROM tracks ORDER BY 3, 2 DESC; You can sort the data using the column num instead of its name

DISTINCT

it allows you to remove the duplicate rows in the result
  • SELECT DISTINCT city FROM customers ORDER BY city; It will keep only one of each duplicate city

Notes for the day

  • We dont add ; to a line start with . like .help
  • In SQLITE it considers NULL is the smallest value in the data when it comes to sorting ASC make it on the top, DESC make it on the bottom

Day 2

Fundamental

WHERE

here we put some conditions and if it is true this would be returned
  • SELECT name, milliseconds, bytes, albumid FROM tracks WHERE albumid = 1; and you can put more than one condition
  • SELECT name, milliseconds, bytes, albumid FROM tracks WHERE albumid = 1 AND milliseconds > 250000;
  • SELECT name, albumid, composer FROM tracks WHERE composer LIKE '%Smith%' ORDER BY albumid; any compaser contains Smith
  • SELECT name, milliseconds, bytes, albumid FROM tracks WHERE albumid IN (1, 2, 3);
  • SELECT name, milliseconds, bytes, albumid FROM tracks WHERE albumid NOT BETWEEN 1 AND 3;
  • SELECT BillingAddress, BillingCity, Total FROM invoices WHERE Total > 5 AND (BillingCity = 'New York' OR BillingCity = 'Chicago') ORDER BY Total;
  • SELECT Name, Composer FROM tracks WHERE Composer IS NULL ORDER BY Name;
  • SELECT Name, Composer FROM tracks WHERE Composer IS NOT NULL ORDER BY Name;

LIMIT

when want to filter your results with showing only some raws and you need to use it always with ORDER BY
  • SELECT trackid, name, milliseconds FROM tracks ORDER BY milliseconds ASC LIMIT 5;
  • SELECT trackid, name, milliseconds FROM tracks ORDER BY milliseconds DESC LIMIT 1 OFFSET 0; gets you the highest mill in the whole list since OFFSET starts counting from 0

JOIN

we use this one when you have to tables and they connected so you wanna filter using the two of them
  • INNER JOIN
    • SELECT users.name, orders.product FROM users INNER JOIN orders ON users.id = orders.user_id; if there's no value or NULL it won't give it to you
  • LEFT JOIN
    • SELECT users.name, orders.product FROM users LEFT JOIN orders ON users.id = orders.user_id; Shows everything from the left table (first one) even if there's no match in the second even NULL
